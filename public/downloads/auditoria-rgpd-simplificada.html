Como usar esta auditoria

Objetivo: Verificar a conformidade da sua empresa com o RGPD e identificar gaps que precisam de correção.

Tempo estimado: 4-6 horas | Dificuldade: Intermédia

Resultado: Plano de ação para conformidade RGPD com prioridades claras.

⚖️ Aviso Legal

Esta auditoria é uma ferramenta informativa. Não substitui aconselhamento jurídico especializado. Recomendamos consultar um advogado especializado em proteção de dados para questões complexas.

📋 Instruções

Marque cada item como ✓ (conforme), ✗ (não conforme) ou N/A (não aplicável). Priorize ações corretivas: OBRIGATÓRIA → IMPORTANTE → RECOMENDADA.

📊 1. Inventário e Mapeamento de Dados

  • Inventário completo de todos os dados pessoais tratados pela empresa OBRIGATÓRIA
  • Identificação de todas as categorias de titulares (clientes, funcionários, fornecedores) OBRIGATÓRIA
  • Mapeamento dos fluxos de dados (origem, tratamento, destino) IMPORTANTE
  • Identificação de dados sensíveis (saúde, origem racial, convicções, etc.) OBRIGATÓRIA
  • Localização física e lógica onde os dados são armazenados IMPORTANTE
  • Periodicidade de atualização do inventário de dados RECOMENDADA

⚖️ 2. Base Legal para Tratamento

  • Base legal claramente identificada para cada tipo de tratamento OBRIGATÓRIA
  • Consentimentos documentados quando essa é a base legal utilizada OBRIGATÓRIA
  • Mecanismo para retirar consentimento facilmente implementado OBRIGATÓRIA
  • Contratos atualizados com cláusulas de proteção de dados IMPORTANTE
  • Avaliação de interesses legítimos quando aplicável IMPORTANTE
  • Processo para verificar a validade dos consentimentos existentes RECOMENDADA

🛡️ 3. Direitos dos Titulares

  • Processo definido para responder a pedidos de acesso aos dados OBRIGATÓRIA
  • Procedimento para retificação de dados pessoais implementado OBRIGATÓRIA
  • Capacidade técnica para apagar dados ("direito ao esquecimento") OBRIGATÓRIA
  • Mecanismo para limitar o tratamento de dados quando solicitado IMPORTANTE
  • Funcionalidade de portabilidade de dados implementada IMPORTANTE
  • Procedimento para oposição ao tratamento de dados IMPORTANTE
  • Tempo de resposta conforme (1 mês, prorrogável por mais 2) OBRIGATÓRIA
  • Formulário ou canal específico para exercício de direitos RECOMENDADA

🔒 4. Medidas Técnicas e Organizativas

  • Encriptação implementada para dados sensíveis OBRIGATÓRIA
  • Controlo de acessos baseado em necessidade de conhecer OBRIGATÓRIA
  • Autenticação multifator para sistemas com dados pessoais IMPORTANTE
  • Backup regular e seguro de dados pessoais IMPORTANTE
  • Política de retenção de dados definida e implementada OBRIGATÓRIA
  • Procedimentos de eliminação segura de dados IMPORTANTE
  • Logs de acesso e alterações em dados pessoais RECOMENDADA
  • Testes regulares das medidas de segurança implementadas RECOMENDADA

🌍 5. Transferências Internacionais

  • Mapeamento de todas as transferências de dados para países terceiros OBRIGATÓRIA
  • Verificação de decisões de adequação para países de destino OBRIGATÓRIA
  • Cláusulas contratuais tipo (CCT) implementadas quando necessário OBRIGATÓRIA
  • Avaliação de impacto para transferências de alto risco IMPORTANTE
  • Certificações ou códigos de conduta verificados RECOMENDADA
  • Revisão regular das garantias para transferências internacionais RECOMENDADA

📋 6. Avaliação de Impacto (AIPD)

  • Critérios definidos para quando fazer uma AIPD IMPORTANTE
  • AIPD realizada para tratamentos de alto risco OBRIGATÓRIA
  • Consulta prévia à CNPD quando aplicável OBRIGATÓRIA
  • Documentação das medidas mitigadoras implementadas IMPORTANTE
  • Revisão periódica das AIPD existentes RECOMENDADA

📝 7. Registos e Documentação

  • Registo das atividades de tratamento mantido e atualizado OBRIGATÓRIA
  • Políticas de proteção de dados documentadas IMPORTANTE
  • Procedimentos de violação de dados documentados OBRIGATÓRIA
  • Contratos com subcontratantes incluem cláusulas RGPD OBRIGATÓRIA
  • Registo de formações em proteção de dados RECOMENDADA
  • Documentação das bases legais e avaliações realizadas IMPORTANTE

🚨 8. Violações de Dados

  • Procedimento de resposta a violações de dados implementado OBRIGATÓRIA
  • Capacidade de notificar a CNPD em 72 horas OBRIGATÓRIA
  • Processo para notificar titulares afetados quando necessário OBRIGATÓRIA
  • Registo de todas as violações de dados (independentemente da notificação) IMPORTANTE
  • Contactos de emergência e responsabilidades definidas IMPORTANTE
  • Simulações regulares de cenários de violação RECOMENDADA

👨‍💼 9. Governance e Responsabilização

  • Responsável designado pela proteção de dados (quando obrigatório DPO) OBRIGATÓRIA
  • Formação regular em proteção de dados para colaboradores IMPORTANTE
  • Revisão anual das políticas de proteção de dados IMPORTANTE
  • Auditoria interna periódica de conformidade RGPD RECOMENDADA
  • Canal para reportar preocupações sobre proteção de dados RECOMENDADA
  • Demonstração de conformidade preparada para auditorias IMPORTANTE

🌐 10. Transparência e Informação

  • Política de privacidade clara e acessível publicada OBRIGATÓRIA
  • Informação sobre tratamentos fornecida no momento da recolha OBRIGATÓRIA
  • Contactos do responsável pelo tratamento facilmente encontráveis OBRIGATÓRIA
  • Informação sobre direitos dos titulares disponível OBRIGATÓRIA
  • Avisos sobre cookies e tecnologias similares IMPORTANTE
  • Linguagem simples e compreensível utilizada nas comunicações RECOMENDADA

Próximos Passos: Priorize ações corretivas (OBRIGATÓRIA → IMPORTANTE → RECOMENDADA)

Prazo de Conformidade: Itens obrigatórios devem ser corrigidos imediatamente

Recomendação: Revise esta auditoria a cada 12 meses ou após mudanças significativas no tratamento de dados
