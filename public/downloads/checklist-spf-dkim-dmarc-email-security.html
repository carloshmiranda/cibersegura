O que são SPF, DKIM e DMARC?

SPF: Define quais servidores podem enviar email pelo seu domínio

DKIM: Assina digitalmente os emails para verificar autenticidade

DMARC: Liga SPF e DKIM, define o que fazer com emails suspeitos

Tempo estimado: 30-60 minutos | Dificuldade: Básica a Intermédia

Resultado: Domínio protegido contra spoofing e melhor deliverability dos emails

📋 Como usar este checklist

1. Implemente SPF primeiro, depois DKIM, e finalmente DMARC

2. Teste cada configuração antes de avançar para o próximo passo

3. Comece com políticas suaves e endureça gradualmente

4. Marque cada item como ✓ (concluído) ou ✗ (pendente)

🔍 1. Preparação e Avaliação Inicial

  • Identificar todos os serviços que enviam email pelo domínio CRÍTICA
    Ex: Google Workspace, Microsoft 365, Mailchimp, Sendinblue, servidor próprio
  • Verificar registos DNS atuais ALTA
    🔧 Ferramentas de verificação:

    • MXToolbox.com • DMARClyzer.com • Google Admin Toolbox

  • Garantir acesso ao painel de gestão DNS CRÍTICA
  • Criar endereço de email para receber relatórios DMARC MÉDIA
    Ex: dmarc@suaempresa.pt ou security@suaempresa.pt

📧 2. Configuração SPF (Sender Policy Framework)

  • Criar registo TXT SPF na raiz do domínio (@) CRÍTICA
    Google Workspace:
    v=spf1 include:_spf.google.com ~all
    Microsoft 365:
    v=spf1 include:spf.protection.outlook.com ~all
    Zoho Mail:
    v=spf1 include:zoho.com ~all
    Múltiplos serviços (exemplo):
    v=spf1 include:_spf.google.com include:servers.mcsv.net ~all
  • Verificar sintaxe do registo SPF ALTA
    🔧 Teste com:

    mxtoolbox.com/spf.aspx

  • Testar envio de email após implementação ALTA
  • Aguardar propagação DNS (24-48 horas) antes de endurecer política MÉDIA
    Atenção: Use ~all (softfail) inicialmente. Só mude para -all após testes
  • Mudança para política rígida: ~all → -all ALTA

🔐 3. Configuração DKIM (DomainKeys Identified Mail)

  • Ativar DKIM no serviço de email CRÍTICA
    Google Workspace:
    Admin Console → Apps → Google Workspace → Gmail → Authenticate email → Generate new record
    Microsoft 365:
    Security.microsoft.com → Email & collaboration → Policies & rules → DKIM
  • Copiar registos CNAME gerados pelo serviço CRÍTICA
    Exemplo típico: google._domainkey.suaempresa.pt
  • Adicionar registos CNAME ao DNS CRÍTICA
  • Verificar configuração DKIM ALTA
    🔧 Teste com:

    mxtoolbox.com/dkim.aspx (usar o seletor correto, ex: "google")

  • Confirmar que emails enviados têm assinatura DKIM ALTA
    Verificar headers dos emails enviados para confirmar "DKIM-Signature:"

🛡️ 4. Configuração DMARC (3 Fases)

Fase 1: Monitorização (Primeira semana)

  • Criar registo TXT DMARC para monitorização CRÍTICA
    _dmarc.suaempresa.pt TXT "v=DMARC1; p=none; rua=mailto:dmarc@suaempresa.pt; ruf=mailto:dmarc@suaempresa.pt; fo=1"
  • Verificar que relatórios DMARC estão a ser recebidos ALTA
  • Analisar relatórios para identificar fontes legítimas de email ALTA

Fase 2: Quarentena (Após 1-2 semanas)

  • Atualizar política DMARC para quarentena CRÍTICA
    _dmarc.suaempresa.pt TXT "v=DMARC1; p=quarantine; pct=25; rua=mailto:dmarc@suaempresa.pt"
    Nota: pct=25 aplica a política apenas a 25% dos emails suspeitos
  • Monitorizar relatórios para falsos positivos ALTA
  • Aumentar gradualmente a percentagem (pct=50, pct=75, pct=100) MÉDIA

Fase 3: Rejeição (Após validação completa)

  • Implementar política de rejeição total ALTA
    _dmarc.suaempresa.pt TXT "v=DMARC1; p=reject; rua=mailto:dmarc@suaempresa.pt"
  • Continuar monitorização regular dos relatórios MÉDIA

📊 5. Validação e Testes Finais

  • Testar todos os registos com ferramentas online CRÍTICA
    🔧 Ferramentas de teste completo:

    • dmarclyzer.com • mxtoolbox.com • mail-tester.com

  • Enviar emails de teste para diferentes provedores ALTA
    Testar: Gmail, Outlook/Hotmail, Yahoo, provedor local português
  • Verificar que emails não vão para spam ALTA
  • Confirmar headers de autenticação nos emails recebidos MÉDIA
    Procurar: "Authentication-Results" nos headers dos emails
  • Documentar todas as configurações implementadas MÉDIA

🔄 6. Manutenção Contínua

  • Configurar revisão mensal dos relatórios DMARC ALTA
  • Atualizar registos SPF quando adicionar novos serviços de email CRÍTICA
  • Monitorizar reputação do domínio trimestralmente MÉDIA
    🔧 Ferramentas de monitorização:

    • Google Postmaster Tools • Microsoft SNDS

  • Formar equipa sobre identificação de emails suspeitos ALTA
⚠️ Avisos Importantes:
• Nunca implemente p=reject diretamente sem teste gradual
• Mantenha backups dos registos DNS antes de alterar
• Teste sempre envio de emails após cada mudança
• A propagação DNS pode demorar até 48 horas
